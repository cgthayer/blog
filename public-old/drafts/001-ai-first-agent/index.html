Running an LLM agent may be a lot easier than you imagine. It boils down to 3 lines of code: create a model, create an agent, run the prompt. Here’s a trivial example:

#!/usr/bin/env python3

# example.py
# pip install smolagents[toolkit]
# pip install smolagents[litellm]
# put api keys in a .env file or environment vars

import dotenv
import os
from smolagents import ToolCallingAgent, LiteLLMModel

dotenv.load_dotenv()

if __name__ == "__main__":
    model = LiteLLMModel(
        model_id="anthropic/claude-3-7-sonnet-latest",
        temperature=0.0,
        api_key=os.environ["ANTHROPIC_API_KEY"]
    )
    agent = ToolCallingAgent(tools=[], model=model)
    result = agent.run("What are the days of the week as a JSON array?")
    print("GOT:", result)

You can skip to the bottom and see that the fun happens at: agent.run("What are the days of the week as a JSON array?")

The rest of it is to create an LLM (model) for your agent to use (which we’ll explain that below). This calls a commercial LLM provider (Anthropic) which will need an api key to work.

Prerequisites
#

Before this will work for you, this is written for python so you’ll python, pip, and a couple libraries.

Python3 and Pip3:

  • Linux (ubuntu): sudo apt install -y python3 python3-pip
  • Mac: brew install python
  • If pip and pip3 are missing, try: python3 -m ensurepip --upgrade

Python libraries:

pip3 install smolagents[toolkit]
pip3 install smolagents[litellm]

API Keys:

  • Also, to use a commercial LLM provider (such as Anthropic or OpenAI) you’ll need an API key, which goes in a .env or can be passed in an environment variable:

Conclusion
#

It’s may be easier than you think. We’ve covered a ton of ground. Using LLMs and agent is good for certain things, and terrible at others. When to try this stuff out:

  • NLP
  • adhoc inputs
  • fuzzy problems
  • classifying, ranking, traditional ML stuff

Future:

  • getting structured answers in JSON
  • accessing message history
  • memory
  • multiple agents talking to each other
  • mcp
  • feeding agents the right context (context engineering and rag)
